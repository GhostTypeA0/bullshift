========================================================
  BULLSHIFT DEPLOYMENT CHEAT SHEET (EC2 + S3 + DOMAIN)
========================================================

This is the full checklist for getting BullShift live so we can
log in from two different computers and chat/post in real time.

========================================================
  1. EC2 BACKEND DEPLOYMENT (SPRING BOOT)
========================================================

-- Create EC2 instance
- Region: us-east-2 (or whatever we pick)
- AMI: Amazon Linux 2023 or Ubuntu 22.04
- Instance: t2.micro (free tier)
- Open ports:
    22 (SSH)
    80 (HTTP)
    443 (HTTPS)
    8080 (Spring Boot)
- Attach security group with these rules

-- SSH into EC2
ssh -i key.pem ec2-user@<EC2-IP>

-- Install Java + Git
sudo dnf install java-17-amazon-corretto git -y

-- Clone backend repo
git clone https://github.com/GhostTypeA0/bullshift.git

-- Build backend
cd bullshift/bullshift_backend
./mvnw clean package

-- Run backend (test)
java -jar target/bullshift_backend-0.0.1-SNAPSHOT.jar

-- Backend should now be reachable at:
http://<EC2-IP>:8080/api/users/login

========================================================
  2. NGINX REVERSE PROXY (FOR CLEAN URL + HTTPS)
========================================================

-- Install nginx
sudo dnf install nginx -y
sudo systemctl enable nginx
sudo systemctl start nginx

-- Edit nginx config
sudo nano /etc/nginx/nginx.conf

-- Add reverse proxy block:
server {
    listen 80;
    server_name <your-domain>;

    location / {
        proxy_pass http://localhost:8080;
    }
}

-- Test + reload
sudo nginx -t
sudo systemctl restart nginx

========================================================
  3. FRONTEND DEPLOYMENT (S3 + CLOUDFRONT)
========================================================

-- Create S3 bucket
- Name: bullshift-frontend
- Enable static website hosting
- Upload:
    index.html
    login.html
    posts.html
    chat.html
    js/
    css/
    images/

-- Make bucket public (static hosting requires it)

-- Create CloudFront distribution
- Origin: S3 bucket
- Default root object: index.html
- Enable HTTPS

-- CloudFront URL becomes our frontend URL

========================================================
  4. CONNECT FRONTEND → BACKEND
========================================================

-- Update API URLs in http.js:
const BASE_URL = "https://<your-domain>/api";

-- Update WebSocket URL in chat.js:
const socket = new SockJS("https://<your-domain>/ws");

-- Re-upload frontend files to S3

========================================================
  5. DOMAIN SETUP (ROUTE 53 OR NAMECHEAP)
========================================================

-- Buy domain (Namecheap or Route 53)

-- Create A record:
@ → CloudFront distribution

-- Create CNAME for www:
www → CloudFront distribution

-- Wait for DNS to propagate (5–30 min)

========================================================
  6. FINAL TEST (CLASSROOM)
========================================================

-- Two computers
-- Two different accounts
-- Both open:
https://<your-domain>/

-- Test:
1. Register + login
2. Create posts
3. Like/unlike
4. Send messages
5. Real-time chat works across machines

========================================================
  7. OPTIONAL (PRODUCTION HARDENING)
========================================================

-- Run backend as a service:
sudo nano /etc/systemd/system/bullshift.service

[Service]
User=ec2-user
ExecStart=/usr/bin/java -jar /home/ec2-user/bullshift_backend.jar
Restart=always

-- Start service:
sudo systemctl daemon-reload
sudo systemctl start bullshift
sudo systemctl enable bullshift

  END OF DEPLOYMENT CHEAT SHEET
